HTML

  • HTLM is one of the cornerstone languages used in web development.
  • HTML elements work together to build the structure of a webpage and provide a framework that makes it easy for a user to navigatethe webpage.
  • We use a series of of HTML elements to define the content and framework. These HTML elements are usually composed of an opening tag and a closing ag represented with an angle brackets (< >).
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • HTML elements are case-sensitive. However declarations are not case-sensitive.

CSS

  • a programming language used by developers to define how webpage content is presented to users.
  • CSS defines the colors, aesthetics, visual layout, and organization. We use CSS to define style rules for specific elements of an HTML page and format them using a wide range of style options.
  • Three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • CSS box model, each box consists of mrgins outside the element, and the element's border, padding, and content.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add A-: to add starter code to current working branch. Add command adds modifications in the current working branch, A- indicatesthat we want to add all changes.
  • staging area: informs Git that the changes will be added in the next commit.
  • git commit: command with a commit message that describes what the commit contains. EX. git commit -m "add starter code"
  • git pull origin main: It is important to always do a git pull to make sure that you have the latest version of the project on your local computer before you start to work.
  • git push: pushes the changes we have locally to our remote GitHub branch

Javascript

  • JavaScript is a high-level dynamic programming language that allows users to interact with websites.
  • A variable is a named container that allows us to store data in our code.
  • To declare a variable we use the var keyword, after the var keyword we give the variable a name. Finally we use an assignment operator to assignthe specific data that we want stored in our variable. We always end it with a semicolon
  • Console.log method is used to output a message to the wev console by adding an argument.
  • Control flow is the order in which a computer executes code in a script.
  • JavaScript is read from top to bottom, and order counts. We must be intentional about how we set up code in a JS file.
  • Control flow can be interrupted, however we can use conditional statements and loops to control the order in which code is executed.
  • There are truthy and falsy values in JS
  • values 0,-0.On,"",null,undefined,NaN, or Boolean data type with the value of false are considered falsy.
  • All other values are considered truthy
  • Arrays is a single variable that is used to hold a group of data.
  • Arrays are useful tools for storing multiple items with different data types in a single variable, but not all arrays must have multiple data types.
  • The location of the data item in the arrayis identified with a unique number called an index.
  • Arrays are zero-indexed and sequential.
  • Function is a set of instructions that tells the computer how to perform a certain task.
  • Functions do not automatically execute when the JS file is run, you must call a function by name in order to execute it.
